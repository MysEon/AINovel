# AINovel - 智能小说创作助手

AINovel 是一款专为小说创作者设计的智能写作平台，致力于帮助作者更高效地创作、管理和组织小说内容。无论您是经验丰富的作家还是刚刚起步的新手，AINovel 都能为您的创作之旅提供全方位的支持。

## 为什么选择 AINovel？

### 全面的创作管理
- **项目管理**：轻松管理多个小说项目，实时跟踪创作进度
- **角色塑造**：深度构建角色形象，记录性格特点、背景故事和外貌特征
- **世界观构建**：精心设计小说世界的规则、魔法体系、科技水平和发展时间线
- **场景描绘**：详细记录故事发生地点的地理环境、文化特色和历史背景
- **组织架构**：设计故事中的组织结构、目的和影响力范围

### 智能创作辅助
- **章节管理**：有序组织小说章节，支持大纲规划和内容编写
- **草稿收集**：随时记录创作灵感和片段，支持标签分类管理
- **字数统计**：自动统计项目总字数和各章节字数，帮助控制创作节奏
- **AI 辅助**：集成多种 AI 模型，提供智能创作建议和内容生成

### 用户友好体验
- **直观界面**：简洁清晰的设计，让您专注于创作本身
- **响应式布局**：支持电脑、平板等多种设备，随时随地创作
- **数据安全**：本地存储您的创作内容，保护您的知识产权
- **无缝协作**：支持多设备同步，创作不间断

## Docker 部署

### 环境要求
- Docker 20.10+
- Docker Compose 2.0+

### 快速部署

1. **使用 Docker Compose**
   ```bash
   docker-compose up --build
   ```

2. **访问应用**
   - 前端：http://localhost:3000
   - 后端：http://localhost:8000

### 生产环境部署

1. **修改 docker-compose.yml**
   ```yaml
   version: '3.8'
   services:
     backend:
       build: ./backend
       ports:
         - "8000:8000"
       environment:
         - NODE_ENV=production
       restart: always
     
     frontend:
       build: ./frontend
       ports:
         - "3000:3000"
       environment:
         - NODE_ENV=production
       restart: always
       depends_on:
         - backend
   ```

2. **启动服务**
   ```bash
   docker-compose up -d
   ```

3. **查看日志**
   ```bash
   docker-compose logs -f
   ```

### 数据持久化

为了确保数据安全，建议使用数据卷持久化数据库：

```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    volumes:
      - ./backend:/app
      - ainovel_data:/app/data
    environment:
      - DATABASE_URL=sqlite:///data/ainovel.db
    restart: always

volumes:
  ainovel_data:
```
